# 系统角色访问逻辑说明

## 五种角色定义

| 角色 | 类型 | 说明 |
|------|------|------|
| **系统管理员** | 职员 | 系统最高权限，管理所有功能 |
| **业务员** | 职员 | 负责客户管理和业务跟进 |
| **工程师** | 职员 | 负责技术服务和现场维修 |
| **技术部长** | 职员 | 负责技术团队管理和审核 |
| **客户** | 外部用户 | 注册成功的企业客户 |

---

## 一、职员角色访问逻辑（前四种）

### 1. 系统管理员
- **默认界面**：电脑端桌面版
- **访问入口**：员工登录页面
- **权限**：所有菜单和功能
- **可切换视图**：❌ 不可切换（只能电脑端）
- **角色切换预览**：可以切换到其他四种角色预览

### 2. 业务员
- **默认界面**：电脑端桌面版
- **访问入口**：员工登录页面
- **权限**：客户管理、工单管理、报价管理等（无系统管理权限）
- **可切换视图**：❌ 不可切换（只能电脑端）
- **角色切换预览**：可以切换到其他四种角色预览

### 3. 工程师
- **默认界面**：电脑端桌面版
- **访问入口**：员工登录页面
- **权限**：工单处理、现场服务、设备查看
- **可切换视图**：✅ 可以切换电脑端/手机端
- **角色切换预览**：可以切换到其他四种角色预览
- **手机端功能**：工单处理、签到打卡、现场拍照

### 4. 技术部长
- **默认界面**：电脑端桌面版
- **访问入口**：员工登录页面
- **权限**：技术审核、团队管理、工单分配
- **可切换视图**：✅ 可以切换电脑端/手机端
- **角色切换预览**：可以切换到其他四种角色预览
- **手机端功能**：工单审核、团队管理、数据统计

---

## 二、客户角色访问逻辑

### 客户（外部用户）
- **默认界面**：手机端客户工作台
- **访问入口**：客户登录页面 `/customer-login`
- **登录方式**：手机号 + 短信验证码
- **权限**：
  - 查看所属企业设备
  - 发起服务工单
  - 查看工单进度
  - 查看服务历史
  - 下载服务报告
  - 评价服务
- **可切换视图**：❌ 不可切换（只能手机端）

### 客户访问流程
```
客户访问系统
    ↓
未登录 → 跳转客户登录页 /customer-login
    ↓
输入手机号 → 获取验证码 → 登录
    ↓
登录成功 → 进入客户首页 /customer-dashboard
    ↓
可使用功能：
  - 查看设备
  - 发起工单
  - 查看进度
  - 评价服务
```

## 三、角色切换下拉框逻辑

### 位置
系统右上角设置按钮旁边

### 功能
方便内部员工快速切换不同角色视角，预览不同角色的系统界面和功能

### 切换逻辑

| 切换到角色 | 跳转页面 | 视图模式 | 说明 |
|-----------|---------|---------|------|
| 系统管理员 | /dashboard | 电脑端 | 管理员桌面版 |
| 业务员 | /dashboard | 电脑端 | 业务员桌面版 |
| 工程师 | /dashboard 或 /staff-mobile-workspace | 根据当前视图 | 可在电脑端和手机端切换 |
| 技术部长 | /dashboard 或 /staff-mobile-workspace | 根据当前视图 | 可在电脑端和手机端切换 |
| 客户 | /customer-workspace | 手机端 | 客户工作台预览（内部预览用） |

### 重要说明
- **客户角色切换**：这是给内部员工预览客户视角用的，实际客户需要通过 `/customer-login` 登录后进入 `/customer-dashboard`
- **视图切换按钮**：只有工程师和技术部长显示"电脑端/手机端"切换按钮
- **菜单权限**：切换角色后，左侧菜单根据角色权限自动更新

## 四、登录入口区分

### 员工登录入口
- **地址**：系统首页或 `/login`（如存在）
- **用户**：系统管理员、业务员、工程师、技术部长
- **认证方式**：账号密码（或企业微信扫码）

### 客户登录入口
- **地址**：`/customer-login`
- **用户**：注册成功的企业客户
- **认证方式**：手机号 + 短信验证码

### 客户注册入口
- **地址**：`/contact-register`
- **用户**：新企业客户
- **流程**：天眼查搜索企业 → 填写信息 → 提交审核

## 五、路由守卫逻辑

```javascript
// 路由守卫伪代码
router.beforeEach((to, from, next) => {
  // 1. 公开页面（注册、登录）
  if (to.meta.public) {
    return next()
  }
  
  // 2. 客户专属页面（需要客户登录）
  if (to.meta.requiresCustomerAuth) {
    if (!isCustomerLoggedIn()) {
      saveRedirectUrl(to.fullPath)
      return next('/customer-login')
    }
    if (customerStatus === 'pending') {
      return next('/contact-register?status=pending')
    }
    return next()
  }
  
  // 3. 员工页面（需要员工登录）
  if (!isEmployeeLoggedIn()) {
    return next('/login')
  }
  
  // 4. 检查角色权限
  if (!hasPermission(currentRole, to.path)) {
    return next('/403')
  }
  
  next()
})
```

## 九、注意事项

1. **客户角色切换**：右上角切换到"客户"角色是内部预览功能，实际客户需要通过 `/customer-login` 登录

2. **视图切换**：只有工程师和技术部长可以切换电脑端/手机端视图

3. **权限隔离**：客户只能看到所属企业的数据，不能看到其他客户的数据

4. **审核状态**：客户注册后需要审核通过才能登录使用

5. **登录状态**：员工登录和客户登录是独立的，互不干扰
