# 报告模板管理功能文档

## 1. 功能概述

报告模板管理模块用于配置Excel格式的服务报告书和结算单模板，支持字段映射、动态表格配置，实现根据模板自动生成PDF报告的功能。

## 2. 核心功能

### 2.1 模板类型

| 类型 | 说明 | 用途 |
|------|------|------|
| `service_report` | 服务报告书 | 工程师服务完成后生成的报告 |
| `settlement` | 结算单 | 服务费用结算单据 |

### 2.2 可用字段列表

#### 工单基础信息
| 字段名 | 标签 | 类型 | 说明 |
|--------|------|------|------|
| `workorderId` | 工单号 | text | 唯一标识 |
| `customerName` | 客户名称 | text | 客户公司名称 |
| `customerAddress` | 客户地址 | text | 服务地址 |
| `deviceSerialNumber` | 设备序列号 | text | 设备SN码 |
| `deviceModel` | 设备型号 | text | 设备型号规格 |
| `assignEngineer` | 指派工程师 | text | 负责工程师姓名 |
| `createTime` | 创建时间 | datetime | 工单创建时间 |
| `finishTime` | 完成时间 | datetime | 服务完成时间 |

#### 报告信息
| 字段名 | 标签 | 类型 | 说明 |
|--------|------|------|------|
| `reportNo` | 报告编号 | text | 报告唯一编号 |
| `serviceDate` | 服务日期 | date | 服务执行日期 |
| `faultDescription` | 故障描述 | textarea | 故障现象描述 |
| `solution` | 处理措施 | textarea | 解决方法说明 |
| `serviceResult` | 服务结果 | text | 服务完成状态 |
| `remark` | 备注 | textarea | 其他备注信息 |

#### 配件信息（数组类型）
| 字段名 | 标签 | 类型 | 说明 |
|--------|------|------|------|
| `parts` | 配件清单 | array | 使用的配件列表 |

#### 费用信息
| 字段名 | 标签 | 类型 | 说明 |
|--------|------|------|------|
| `totalAmount` | 总金额 | number | 费用总计 |
| `materialCost` | 材料费 | number | 配件费用 |
| `laborCost` | 人工费 | number | 人工服务费用 |
| `otherCost` | 其他费用 | number | 其他杂费 |

#### 签字信息
| 字段名 | 标签 | 类型 | 说明 |
|--------|------|------|------|
| `customerSign` | 客户签字 | image | 客户签名图片 |
| `engineerSign` | 工程师签字 | image | 工程师签名图片 |
| `signDate` | 签字日期 | date | 签字日期 |

## 5. 模板数据结构

```typescript
interface Template {
  id: string;                    // 模板ID
  name: string;                  // 模板名称
  type: 'service_report' | 'settlement';  // 模板类型
  description: string;           // 描述
  isDefault: boolean;            // 是否默认
  createTime: string;            // 创建时间
  updateTime: string;            // 更新时间
  excelFile: {                   // Excel文件信息
    name: string;
    size: number;
    type: string;
    data: string;               // Base64编码
    uploadTime: string;
  };
  fieldMappings: FieldMapping[]; // 字段映射配置
  tableConfigs: TableConfig[];   // 表格配置
}

interface FieldMapping {
  field: string;      // 字段名
  cellRef: string;    // 单元格位置，如 'A1'
  type: string;       // 数据类型
}

interface TableConfig {
  name: string;       // 数据字段名
  startCell: string;  // 起始单元格
  columns: Column[];  // 列配置
}

interface Column {
  field: string;      // 数据字段
  header: string;     // 表头名称
}
```

## 6. 使用流程

### 6.1 创建模板流程

1. 进入"报告模板管理"页面
2. 点击"新增模板"按钮
3. 填写模板信息：
   - 模板名称
   - 模板类型（服务报告书/结算单）
   - 描述（可选）
4. 上传Excel模板文件（.xlsx/.xls）
5. 保存模板

### 6.2 配置字段映射流程

1. 在模板列表中找到目标模板
2. 点击"配置"按钮
3. 添加字段映射：
   - 选择字段（如：工单号、客户名称等）
   - 输入单元格位置（如：A1, B2）
   - 选择数据类型（文本/日期/数字/图片等）
4. 配置动态表格（可选）：
   - 选择数据字段（如：配件清单）
   - 设置起始单元格
   - 配置列映射
5. 保存配置

### 6.3 使用模板生成报告（代码示例）

```javascript
// 1. 获取默认模板
import { getDefaultTemplate, TemplateType } from '../stores/reportTemplateStore.js'
import { generateExcelFromTemplate } from '../utils/excelUtils.js'

const template = getDefaultTemplate(TemplateType.SERVICE_REPORT)

// 2. 准备工单数据
const workorderData = {
  workorderId: 'WO2024001',
  customerName: '某某公司',
  customerAddress: '某某地址',
  assignEngineer: '张工程师',
  serviceDate: new Date(),
  faultDescription: '设备无法启动',
  solution: '更换电源模块',
  serviceResult: '已解决',
  parts: [
    { name: '电源模块', model: 'PM-220V', quantity: 1 },
    { name: '保险丝', model: 'F10A', quantity: 2 }
  ],
  customerSign: 'data:image/png;base64,...'  // 签名图片
}

// 3. 生成Excel
try {
  const excelDataUrl = await generateExcelFromTemplate(template, workorderData)
  // 使用生成的Excel DataURL进行下载或预览
} catch (error) {
  console.error('生成报告失败:', error)
}
```

## 9. 技术实现细节

### 9.1 xlsx库动态导入

为避免SSR问题，使用Vite动态导入：

```javascript
let XLSX = null

const getXLSX = async () => {
  if (!XLSX) {
    const module = await import('xlsx/dist/xlsx.full.min.js')
    XLSX = module.default || module
  }
  return XLSX
}
```

### 9.2 单元格引用转换

支持Excel单元格引用（如A1）与行列索引的相互转换：

```javascript
// A1 -> {row: 0, col: 0}
cellRefToIndex('A1')  // {row: 0, col: 0}

// {row: 0, col: 0} -> A1
indexToCellRef(0, 0)  // 'A1'
```

### 9.3 数据类型处理

支持多种数据类型的自动格式化：

| 类型 | 处理方式 |
|------|----------|
| `text` | 直接转为字符串 |
| `textarea` | 多行文本，保留换行 |
| `date` | 格式化为 yyyy-mm-dd |
| `datetime` | 格式化为 yyyy-mm-dd hh:mm:ss |
| `number` | 保留2位小数 |
| `image` | 特殊处理，预留图片插入位置 |
| `array` | 动态表格填充 |

## 10. 注意事项

1. **默认模板保护**：默认模板不能被删除，但可以被修改
2. **Excel文件格式**：仅支持 .xlsx 和 .xls 格式
3. **图片插入限制**：当前使用xlsx库不直接支持图片插入，如需完整图片支持可集成exceljs
4. **数据安全**：Excel文件以Base64格式存储在内存中，实际生产环境建议存储到服务器
5. **浏览器兼容性**：推荐使用Chrome、Firefox、Edge等现代浏览器
