## 一、功能概述

### 1.1 定位

字段模板管理是**动态表单系统的核心模块**，用于：

- 预定义不同业务场景的表单字段组合
- 实现工单类型的差异化字段配置
- 支持客户级别的字段定制
- 版本控制与模板复用

### 1.2 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                    字段模板管理体系                          │
├─────────────────────────────────────────────────────────────┤
│  标准模板库 → 客户定制 → 工单实例                            │
│                                                             │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐             │
│  │维修标准模板│ → │客户A定制 │ → │工单#001  │             │
│  │安装标准模板│    │客户B定制 │    │工单#002  │             │
│  │巡检标准模板│    └──────────┘    └──────────┘             │
│  └──────────┘                                               │
└─────────────────────────────────────────────────────────────┘
```

## 二、核心概念

### 2.1 字段模板 (FieldTemplate)

```javascript
{
  id: 'template_repair_standard',     // 模板ID
  code: 'REPAIR_STANDARD',            // 模板编码（唯一）
  name: '标准维修字段模板',            // 模板名称
  description: '标准维修工单字段配置',  // 描述
  
  // 适用范围
  scope: {
    workorderTypes: ['维修'],          // 适用工单类型
    scenes: ['现场维修', '返厂维修'],   // 适用场景
    customers: []                       // 特定客户（空表示通用）
  },
  
  // 字段列表
  fields: [
    { fieldDefId: 'field_title', sortOrder: 1, colSpan: 12 },
    { fieldDefId: 'field_customer', sortOrder: 2, colSpan: 12 }
  ],
  
  // 布局配置
  layout: {
    type: 'grid',      // 布局类型：grid/flex
    columns: 24,       // 栅格列数
    gutter: 16         // 间距
  },
  
  // 节点字段映射（工作流各阶段显示字段）
  nodeFieldMapping: {
    'CREATED': ['title', 'customer', 'priority'],
    'DISPATCHED': ['engineer', 'location'],
    'COMPLETED': ['laborCost', 'partsCost']
  },
  
  metadata: {
    version: 1,        // 版本号
    isDefault: true,   // 是否默认模板
    status: 'active'   // 状态：draft/active/deprecated
  }
}
```

### 2.2 客户定制 (CustomerConfig)

```javascript
{
  customerId: 'customer_001',
  templateId: 'template_repair_standard',
  customizations: {
    // 移除字段
    removeFields: ['field_rating'],
    
    // 修改字段
    modifyFields: [
      {
        fieldDefId: 'field_priority',
        changes: { required: false, defaultValue: 'normal' }
      }
    ],
    
    // 添加字段
    addFields: [
      { fieldDefId: 'field_custom_field', sortOrder: 99 }
    ]
  }
}
```

## 三、使用流程

### 3.1 进入模板管理

访问地址：`http://localhost:3000/field-template-manager`

### 3.2 创建新模板

**步骤1：基本信息**

- 点击「新建模板」
- 填写模板名称（如：VIP客户维修模板）
- 填写模板编码（如：REPAIR\_VIP，必须大写字母开头）
- 选择适用工单类型（可多选）
- 选择适用场景（可多选）

**步骤2：设计字段**

- 点击「设计」进入字段设计器
- 从字段库拖拽字段到画布
- 调整字段顺序和布局
- 配置字段属性（必填、默认值等）

**步骤3：配置节点映射**

- 设置各工作流节点显示的字段
- 例如：创建节点只显示基本信息，完成节点显示费用信息

**步骤4：发布**

- 点击「发布」使模板生效
- 可设为默认模板（同类型工单自动使用）

### 3.3 模板设计器使用

访问地址：`http://localhost:3000/field-designer?templateId=xxx`

| 功能   | 操作                   |
| ---- | -------------------- |
| 添加字段 | 左侧面板选择字段，拖拽到画布       |
| 删除字段 | 点击字段右上角删除图标          |
| 调整顺序 | 拖拽字段上下移动             |
| 设置列宽 | 点击字段设置 colSpan（1-24） |
| 字段配置 | 点击字段展开属性面板           |

## 四、字段类型详解

### 4.1 分类总览

| 分类       | 颜色标识 | 字段数量 | 用途             |
| -------- | ---- | ---- | -------------- |
| **基础字段** | 蓝色   | 9个   | 文本、数字、日期等基础输入  |
| **选择字段** | 绿色   | 6个   | 单选、多选、开关等选择控件  |
| **业务字段** | 橙色   | 11个  | 客户、设备、签名等业务相关  |
| **高级字段** | 紫色   | 6个   | 富文本、表格、计算等复杂场景 |

### 4.2 基础字段 (Basic Fields)

## 九、常见问题

**Q：模板发布后可以修改吗？**
A：可以，修改后会自动创建新版本，不影响已创建的工单。

**Q：客户定制和模板的关系？**
A：客户定制是基于模板的差异化配置，优先级高于模板默认配置。

**Q：如何删除字段？**
A：在模板设计器中移除字段引用，字段定义本身保留（可能被其他模板使用）。

**Q：多个模板匹配同个工单怎么办？**
A：优先使用客户专属模板，其次按场景精确匹配，最后使用默认模板。

**列宽计算：**

- 总宽度 = 24列
- 字段宽度 = (colSpan / 24) × 100%

### 15.8 使用流程

#### 新建字段流程

**步骤1：填写基础信息**

1. 选择规则类型
2. 输入规则值（如需要）

1. 点击「保存」

### 15.9 最佳实践

#### 字段命名规范

**字段编码：**

- 使用小写字母开头
- 单词间用下划线分隔（snake\_case）
- 示例：`customer_name`、`service_date`、`priority_level`

**字段名称：**

- 简洁明了
- 避免使用缩写
- 示例：`客户名称`、`服务日期`、`优先级`

#### 字段复用原则

1. **通用字段优先**
   - 优先使用系统已有字段
   - 避免重复创建相似字段
2. **字段粒度适中**
   - 不要过细（如：first\_name、last\_name 合并为 name）
   - 不要过粗（如：地址拆分为省、市、区、街道）
3. **类型选择准确**
   - 日期用 date，不要用 text
   - 金额用 number，不要用 text
   - 选项用 select，不要用 text

#### 验证规则配置

1. **必填规则**
   - 核心字段设为必填
   - 避免过多必填造成使用困难
2. **长度限制**
   - 文本字段设置合理最大长度
   - 手机号、身份证号等设置精确长度
3. **格式验证**
   - 邮箱、手机号使用内置格式验证
   - 自定义格式使用正则表达式

### 15.10 常见问题

**Q：字段编码可以修改吗？**
A：创建后不可修改。如需修改，建议删除后重新创建，或复制字段后修改编码。

**Q：删除字段会影响已有工单吗？**
A：会影响。删除字段后，已有工单中该字段的数据将无法显示。建议先确认无使用后再删除。

**Q：字段类型可以修改吗？**
A：编辑时不可修改类型。如需更换类型，建议创建新字段后迁移数据。

**Q：如何批量创建字段？**
A：使用导入功能，准备 JSON 文件批量导入。

**Q：字段在模板中如何使用？**
A：在字段模板管理中选择「设计」，从左侧字段列表拖拽字段到画布。
