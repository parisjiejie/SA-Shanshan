# 售后管理系统 - 角色对应关系说明

> ⚠️ **重要提示**：此文档记录了系统中所有角色对应关系，修改前请务必参考，避免角色错乱！

---

## 一、员工角色体系

### 1.1 登录账号（StaffLogin.vue）

| 用户名 | 密码 | role字段 | 显示名称 | userInfo.role |
|--------|------|----------|----------|---------------|
| admin | 123456 | admin | 管理员 | 系统管理员 |
| sales | 123456 | sales | 业务员 | 业务员 |
| engineer | 123456 | engineer | 工程师 | 工程师 |
| techLead | 123456 | techLead | 技术部长 | 技术主管 |

**存储数据结构：**
```javascript
// staffAuth (用于系统认证和权限判断)
{
  username: 'engineer',
  name: '工程师',
  role: 'engineer',  // 用于菜单权限判断
  loginTime: '2024-01-15T10:30:00.000Z'
}

// userInfo (用于手机端工作台显示)
{
  name: '工程师',
  role: '工程师',  // 中文角色名，用于显示和审批权限判断
  department: '技术服务部'
}
```

### 1.3 手机端视图权限

**可切换手机端的角色：**
- engineer (工程师)
- techLead (技术部长/技术主管)

**不可切换手机端的角色：**
- admin (管理员)
- sales (业务员)

### 1.4 手机端审批权限（StaffMobileWorkspace.vue）

```javascript
// 审批者角色（可审批外勤记录）
isApprover: ['技术主管', '经理', '部门负责人']

// 经理角色
isManager: ['技术主管', '经理']
```

**对应关系：**
| 系统role | userInfo.role | 是否有审批权限 |
|----------|---------------|----------------|
| engineer | 工程师 | ❌ 否 |
| techLead | 技术主管 | ✅ 是 |

## 二、客户角色体系

### 2.1 登录方式（CustomerLogin.vue）

- 使用手机号 + 短信验证码登录
- 客户数据存储在 customerStore.js

### 2.2 客户状态

| 状态 | 说明 | 可访问页面 |
|------|------|-----------|
| active | 已审核通过 | 客户工作台等所有客户页面 |
| pending | 待审核 | 注册页面（显示审核中） |
| rejected | 审核被拒绝 | 注册页面（显示被拒绝） |

### 2.3 客户可访问页面

- /customer-workspace (客户工作台)
- /customer-workorder-list (工单列表)
- /customer-workorder-detail (工单详情)
- /customer-device-list (设备列表)
- /customer-profile (个人资料)
- /asset-detail (设备详情)

**存储数据结构：**
```javascript
// customerAuth
{
  customerId: 'CUS001',
  phone: '13800138001',
  name: '张经理',
  companyId: 'COMP001',
  companyName: '上海某机械有限公司',
  status: 'active',
  loginTime: '2024-01-15T10:30:00.000Z'
}
```

## 三、localStorage 存储项汇总

| 键名 | 用途 | 存储位置 | 清除位置 |
|------|------|----------|----------|
| staffAuth | 员工登录状态 | StaffLogin.vue | App.vue handleLogout |
| userInfo | 员工信息显示 | StaffLogin.vue | App.vue handleLogout |
| staffViewMode | 视图模式(desktop/mobile) | App.vue | App.vue handleLogout |
| customerAuth | 客户登录状态 | customerStore.js | customerStore.js logout |

---

## 四、关键文件职责

| 文件 | 职责 |
|------|------|
| StaffLogin.vue | 员工登录，保存 staffAuth 和 userInfo |
| App.vue | 读取 staffAuth，控制菜单显示和视图切换按钮 |
| StaffMobileWorkspace.vue | 读取 userInfo，显示员工信息和判断审批权限 |
| CustomerLogin.vue | 客户登录，使用 customerStore |
| customerStore.js | 管理客户状态和认证 |
| router/index.js | 路由守卫，控制页面访问权限 |

## 五、常见问题排查

### 问题1：登录后显示错误角色
**原因：** App.vue 的 onMounted 没有重新执行
**解决：** StaffLogin.vue 使用 window.location.href 跳转，强制刷新页面

### 问题2：手机端显示错误用户信息
**原因：** StaffMobileWorkspace.vue 从 userInfo 读取，但登录时未保存
**解决：** StaffLogin.vue 登录时同时保存 staffAuth 和 userInfo

### 问题3：审批权限判断错误
**原因：** StaffMobileWorkspace.vue 使用中文角色名判断，但保存的 role 字段不对
**解决：** 确保 roleMapping 映射正确
- engineer → 工程师
- techLead → 技术主管
