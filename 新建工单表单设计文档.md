# 新建工单表单设计文档

## 一、表单字段

### 1. 基本信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 工单号 | 自动生成 | - | 格式 `WO{YYYYMMDD}{序号}`，不可编辑，创建时自动生成 |
| 类型 | 选择 | 是 | 一级：服务工单/安装工单；二级（服务工单下）：维修/试加工/改装 |
| 创建时间 | 不显示 | - | 后台记录 `createTime`，前端表单不展示 |

### 2. 客户信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 客户 | 搜索选择 | 是 | 输入关键词搜索客户公司，从下拉列表中选择 |
| 电话 | 自动带入 | - | 选择客户后自动填充，不可编辑 |
| 地址 | 自动带入 | - | 选择客户后自动填充，不可编辑 |

### 3. 设备信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 型号 | 级联选择 | 否 | 选择客户后，加载该客户名下的设备型号列表供选择 |
| 序列号 | 级联选择 | 是 | ① 已选型号 → 加载该型号下的SN号列表；② 未选型号 → 加载该客户名下所有SN号 |
| 保修状态 | 自动带入 | - | 根据型号+SN定位设备，自动显示（保内/保外/过保），不可编辑 |

### 4. 故障描述

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 故障描述 | 填写 + 标签快选 | 是 | 支持手动输入；提供常见故障标签，点击自动填入描述框 |

#### 常见故障标签（通用，5个）

| 标签 | 填入内容 |
|------|----------|
| 无法启动 | 设备无法正常启动，通电后无响应 |
| 异响 | 设备运行过程中出现异常响声 |
| 漏油 | 设备存在漏油现象，需检查密封件 |
| 精度异常 | 设备加工精度超出允许偏差范围 |
| 过热报警 | 设备运行中触发过热报警保护 |

## 二、联动逻辑

```
选择客户 ──┬──→ 自动带入：电话、地址
           │
           └──→ 加载：该客户名下的设备型号列表
                  │
                  ├─ 选择型号 ──→ 加载：该型号下的SN号列表
                  │                    │
                  │                    └─ 选择SN ──→ 自动带入：保修状态
                  │
                  └─ 未选型号 ──→ 加载：该客户名下所有SN号
                                      │
                                      └─ 选择SN ──→ 自动带入：保修状态 + 回填型号
```

## 五、表单交互规则

1. **客户选择**：下拉框支持输入搜索过滤，选择后自动填充电话、地址
2. **型号选择**：选择客户后才能选择型号，型号为非必填
3. **序列号选择**：选择客户后才能选择序列号，为必填项
   - 已选型号 → 只显示该型号下的SN
   - 未选型号 → 显示该客户所有SN
   - 选择SN后自动回填型号（如果型号未选）
4. **保修状态**：选择SN后自动显示，不可手动修改
5. **故障标签**：点击标签将描述追加到故障描述框，可多次点击追加，也可手动编辑
6. **工单号**：提交时自动生成，不在表单中显示

## 六、与现有代码的关系

- 模拟数据将放置在 `workorderFlowStore.js` 中，与现有 mock 数据统一管理
- 客户和设备数据可抽取为独立的 store（如 `customerStore.js`、`assetStore.js`），供多个页面复用
- 新建工单表单的 UI 修改集中在 `Workorder.vue` 的 `dialogVisible` 弹窗区域
- 工单创建后，客户信息、设备信息、保修状态需保存到工单记录中
