# 售后管理系统 - 5分钟快速部署指南

> 无需后端，纯前端演示部署方案。适合紧急演示、客户展示场景。

## 快速部署步骤（5分钟完成）

### 第1步：构建项目（1分钟）

在项目根目录执行：

```bash
# 1. 确保使用生产环境配置（已配置好Mock模式）
# .env.production 中 VITE_USE_MOCK=true

# 2. 安装依赖（如已安装可跳过）
npm install

# 3. 生产环境构建
npm run build
```

构建完成后，会生成 `dist/` 目录。

---

### 第2步：创建 OSS 存储桶（2分钟）

1. 登录 [阿里云 OSS 控制台](https://oss.console.aliyun.com/)
2. 点击「创建 Bucket」：

| 配置项 | 建议值 |
|--------|--------|
| Bucket 名称 | `yourcompany-demo`（全局唯一，小写字母+数字+短横线） |
| 地域 | 选择离你最近的（如华东1-杭州） |
| 存储类型 | 标准存储 |
| 读写权限 | **公共读**（必须） |
| 其他选项 | 保持默认 |

3. 点击「确定」创建

---

### 第3步：配置静态网站托管（1分钟）

1. 进入刚创建的 Bucket
2. 左侧菜单 → **基础设置** → **静态页面**
3. 配置：
   - 默认首页：`index.html`
   - 默认404页：`index.html`（支持前端路由）
4. 点击「保存」

---

### 第4步：上传文件（1分钟）

#### 方式A：使用阿里云控制台（推荐新手）

1. 进入 Bucket → **文件管理**
2. 点击「上传文件」
3. 选择 `dist` 文件夹内的所有文件和文件夹
4. 等待上传完成

#### 方式B：使用 ossutil 工具（批量上传）

```bash
# 1. 下载并安装 ossutil
# https://help.aliyun.com/document_detail/120075.html

# 2. 配置 ossutil
ossutil config

# 3. 上传 dist 目录
ossutil cp -r dist/ oss://yourcompany-demo/ -f
```

---

### 第5步：访问演示系统

上传完成后，通过以下地址访问：

```
http://yourcompany-demo.oss-cn-hangzhou.aliyuncs.com
```

> 注意：将 `yourcompany-demo` 替换为你的 Bucket 名称，`oss-cn-hangzhou` 替换为你的地域。

---

## 可选：绑定自定义域名

### 步骤：

1. **域名解析**：
   - 登录阿里云域名控制台
   - 添加 CNAME 记录：
     - 主机记录：`demo`（或你想要的子域名）
     - 记录值：`yourcompany-demo.oss-cn-hangzhou.aliyuncs.com`

2. **OSS 绑定域名**：
   - 进入 Bucket → **传输管理** → **域名管理**
   - 点击「绑定域名」
   - 输入你的域名，如 `demo.yourcompany.com`
   - 开启「CDN 加速」（推荐）

3. **HTTPS 配置**（可选）：
   - 在 CDN 控制台申请免费 SSL 证书
   - 或上传已有证书

4. **访问**：
   ```
   https://demo.yourcompany.com
   ```

## 常见问题

### Q1: 页面显示空白？

**解决**：检查 `vite.config.js` 是否有 `base` 配置：

```javascript
export default defineConfig({
  base: './',  // 使用相对路径
  // ...
})
```

### Q2: 刷新页面404？

**解决**：确保 OSS 静态页面配置中设置了默认404页为 `index.html`

### Q3: API 请求失败？

**解决**：检查浏览器控制台：
1. 确认 `.env.production` 中 `VITE_USE_MOCK=true`
2. 重新构建项目
3. 清除浏览器缓存

### Q4: 如何修改演示数据？

编辑 `src/mock/index.js` 文件，修改对应的数据生成规则，然后重新构建上传。
