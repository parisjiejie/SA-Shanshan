### [1] 待分配 (`pending_assign`)

| 角色 | 端 | 页面位置 | 操作按钮 | 条件 |

## 三、修改工单流程时的强制检查清单

1. [ ] `canCreateWorkorder` 是否正确（admin/assistant/customer）
2. [ ] `getVisibleWorkorders` 各角色返回范围 → 详见 `sa-role-permission`
3. [ ] 每个状态节点的 `getAvailableActions` 返回的操作是否正确
4. [ ] 状态流转时的通知目标角色 → 详见 `sa-notification`
5. [ ] 超时通知的 `targetRole` → 详见 `sa-notification`
6. [ ] Mock 数据的工单状态能覆盖全部 7+1 个节点
7. [ ] 各 UI 页面的按钮显示条件与角色权限一致
