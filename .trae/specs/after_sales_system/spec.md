# 售后系统 - 产品需求文档

## Overview

* **Summary**: 为大型产业机械设备代理销售及服务公司构建一套完整的售后管理平台，整合客户服务全流程，提升服务效率与客户满意度。系统覆盖客户管理、出货机器管理、工单处理、现场服务、配件销售等核心业务场景。

* **Purpose**: 解决售后服务标准化、现场作业数字化、设备管理精细化、数据决策可视化等问题，提升客户服务质量和企业运营效率。

* **Target Users**: 
  * 公司内部员工，包括管理人员、客服人员、工程师、配件管理人员等
  * **联系人（客户员工）**: 通过企业微信服务号或微信小程序访问系统，发起工单、查看设备状态、接收服务通知

## Goals

* 售后服务标准化：统一工单流程、服务规范及报告格式

* 现场作业数字化：通过移动端实现打卡、拍照、签字无纸化

* 设备管理精细化：实现"一机一码"全生命周期追踪

* 数据决策可视化：通过数据看板辅助管理层决策

* 基于企业微信服务号生态，建立统一的客户服务入口与标准化流程

## Non-Goals (Out of Scope)

* 销售管理功能（如订单处理、报价管理等）

* 财务管理功能（如发票管理、收款管理等）

* 人力资源管理功能（如考勤、绩效等）

* 与第三方ERP系统的深度集成

## Background & Context

* 公司目前采用传统的售后服务方式，存在流程不规范、信息孤岛、数据统计困难等问题

* 随着业务规模的扩大，需要更高效的售后管理系统来支撑业务发展

* 企业微信已成为公司内部沟通的主要工具，可作为系统的统一入口

* 产业机械设备价值高，需要精细化的设备管理和全生命周期追踪

* **基于企业微信服务号以及微信小程序生态，建立统一的客户服务入口与标准化流程，让联系人（客户员工）能够便捷地发起服务请求**

## Functional Requirements

* **FR-1**: 客户管理

  * ***多主体关联***：

    * 数据库设计：`用户表 (User)` 与 `企业表 (Company)` 为 N:N 关系。

    * 逻辑：一个微信用户（OpenID）可绑定多个企业身份。切换企业时，系统上下文（可见设备、工单）随之切换。

  * ***天眼查集成***：

    注册时输入关键词，调用天眼查 API 返回标准企业列表。

    选中后自动填充：统一社会信用代码、注册地址、法人等。

    * 优化*：增加本地缓存，避免重复查询消耗 API 额度。

  - ***注册审核流***：

    **方案：天眼查API搜索选择 + 员工审核（终极防重复方案）**

    * **核心设计原则**：
      * **完全防重复**：客户无法手填企业名称，必须从天眼查标准数据库选择
      * **数据标准化**：所有企业信息来自天眼查，确保数据质量和一致性
      * **简化注册流程**：搜索选择企业 → 自动填充信息 → 填写个人信息 → 提交审核

    * **客户自主注册流程**：
      * **步骤1 - 从天眼查搜索企业**：
        * 客户输入企业名称关键词（如"腾讯"、"阿里巴巴"）
        * 点击搜索按钮，系统调用天眼查API进行企业搜索
        * 显示搜索结果列表，每个企业展示：
          * 企业全称
          * 统一社会信用代码
          * 注册地址
          * 法人代表
          * 企业状态（存续/在业/注销等）
        * 客户从列表中选择正确的企业
        * 如未找到企业，提供"手动输入"备用方案（需更严格审核）
      
      * **步骤2 - 确认企业信息**：
        * 系统自动填充所选企业的全部信息
        * 展示已选企业信息卡片：
          * 企业名称
          * 统一社会信用代码
          * 注册地址
          * 法人代表
          * 企业状态
        * 提供"重新选择"功能，允许客户重新搜索
      
      * **步骤3 - 填写个人信息**：
        * 姓名（必填）：申请人真实姓名
        * 手机号（必填）：用于接收验证码和审核通知，作为登录账号
        * 职位（必填）：申请人在企业中的职位
        * 邮箱（选填）：用于接收系统通知
      
      * **步骤4 - 身份验证**：
        * 手机号短信验证码验证（6位数字，5分钟有效）
        * 防刷机制：1分钟内只能发送1次，每小时最多10次
      
      * **步骤5 - 提交注册**：
        * 阅读并同意《用户注册协议》
        * 提交注册申请
        * 创建"待审核"客户账号
        * 如企业未在系统中存在，同时创建"待审核"企业档案
        * 发送审核通知给业务员/管理员
        * 客户收到"审核中"状态提示，可查看审核进度
    
    * **天眼查API集成细节**：
      * **API接口**：调用天眼查企业搜索接口（/services/v3/search/sNorV2）
      * **搜索方式**：支持关键词模糊搜索
      * **返回字段**：企业名称、统一社会信用代码、注册地址、法人代表、企业状态、注册资本、成立日期等
      * **数据缓存**：本地缓存搜索结果，避免重复查询消耗API额度
      * **错误处理**：API调用失败时提供友好提示，引导手动输入
      * **备用方案**：天眼查未收录的企业，允许手动输入（标记为"手动录入"，审核更严格）
    
    * **防重复机制（终极方案）**：
      * **天眼查数据源**：所有企业来自天眼查标准数据库，数据权威准确
      * **统一信用代码唯一性**：每个企业有唯一的18位统一社会信用代码
      * **无手填入口**：客户无法随意输入企业名称，必须从列表选择
      * **手动输入备用**：仅当天眼查未收录时才允许手动输入，且需要更严格的审核流程
      * **系统内重复检测**：即使从天眼查选择，仍需检查该企业是否已在系统内注册
    
    * **员工审核流程**：
      * **审核任务分配**：
        * 系统自动分配给对应区域的业务员
        * 或统一进入"客户注册审核"待办池
      
      * **审核内容**：
        * 企业信息审核：核对营业执照、企业名称规范性、信用代码准确性
        * 联系人身份审核：确认申请人为企业员工
        * 重复企业检测：审核界面显示相似企业警告，支持合并到已有企业
      
      * **审核操作**：
        * **通过**：企业和客户账号同时激活，发送通知给客户
        * **拒绝**：填写拒绝原因，客户可修改后重新提交
        * **要求补充信息**：退回给客户补充材料
        * **合并到已有企业**：发现重复时，将客户关联到已有企业
      
      * **审核时效**：建议3个工作日内完成审核
    
    * **审核通过后**：
      * 客户收到短信/微信通知
      * 可登录系统查看设备、发起工单
      * 自动分配专属业务员跟进

  * ***核心功能：***

    客户分级管理（VIP/普通/潜在）

    联系人管理（一家客户可以多联系人）

    已购设备关联

* **FR-2**: 客户与设备档案 (Customer & Asset)

* **客户 360 视图**：基础信息、联系人树、历史合同、在保设备列表、历史工单、投诉记录。

  设备管理字段：机器序列号、型号、出厂日期、销售日期、保修截止日、当前状态（运行/停机/维修中）是否EL、安装地址等。

  来源：与订单/出货系统(才望子kintone)同步，或安装工单完成后自动创建。

  一机一码：每台设备绑定动态二维码。

  扫码端（客户/工程师）：

    客户扫：查看设备档案、维修记录、一键报修、查看操作手册等。

     工程师扫：查看设备档案、维修记录、一键报修、查看操作手册等。

  * 状态追踪：安装 -> 验收 -> 保修中 -> 出保 -> 报废。

* **FR-3**: 工单管理

  * **安装工单**：

    **触发：**订单系统发货后自动生成，或销售手动创建。

    **关键动作**：扫描"空白二维码" -> 绑定机器序列号 -> 激活设备档案。

    **GPS定位获取位置信息：**将位置信息同时同步设备档案的安装地址中。

    **安装报告：**安装报告填写，自动生成安装报告PDF

    **客户签字：**客户签字三种场景：线上签字的话，签名保存在相应PDF中

  

  ①工程师手机直接签字②下载后发客户签字，然后有工程师将签字报告书PDF上传系统③推送我微信通知，客户手机微信端签字回传。

  **PDF 报告**：内置 HTML 模板引擎，完工后自动填充数据（故障现象、处理过程、更换配件、现场照片等），生成带电子签名的 PDF，支持一键发送给客户。

  **安装列表：支持模糊检索，多条件筛选，排序，列表显示字段以及显示顺序自定义**

* **服务工单**：

  **类型：**维修、改造、巡检。

  **流程：**客户报修/工程师自建 -> 派单 -> 接单 -> 到场打卡 -> 作业对应 -> 服务报告书制作 -> 客户签字（作业报告书）  -> 完工打卡-> 评价-> 结算单制作->客户签字（结算单）。

  **服务报告书：服务**报告填写，自动生成服务报告PDF

  **结算单：**结算单制作完成后，自动生成PDF结算单

  **客户签字：**客户签字三种场景：线上签字的话，签名保存在相应PDF中

  ①工程师手机直接签字②下载后发客户签字，然后有工程师将签字报告书PDF上传系统③推送我微信通知，客户手机微信端签字回传。

  **PDF 报告**：内置 HTML 模板引擎，完工后自动填充数据（故障现象、处理过程、更换配件、现场照片等），生成带电子签名的 PDF，支持一键发送给客户。

  **服务工单列表：支持模糊检索，支持多条件筛选，排序，列表显示字段以及显示顺序自定义**

* **配件销售工单**：

  **配件清单**：工单内嵌配件清单。字段：成本价，销售价，利润率，数量等。

  **配件明细excel数据导入功能**

  **配件报价明细的查询功能：**已经提交给客户的报价单中的配件明细需要可以检索以及列表管理，可多维度查询 例：同一客户多同型号不同时期的价格查询或者不同客户同一型号不同时期的价格查询等

  **流程：**客户询价/工程师自建 -> 派单 -> 接单 -> 制作报价单 -> 内部审核报价 -> 客户确认（报价单） -> 订单/丢单

  **报价单与工单关联** (v2.2新增)：
  * **基于工单的报价单**：报价单必须关联到具体的工单（配件销售工单）
  * **客户信息继承**：选择工单后，客户名称、联系人、联系电话自动从工单继承
  * **双入口创建**：
    - 入口1：在报价单管理页面新建，需选择工单
    - 入口2：在工单详情页直接发起报价单
  * **工单详情展示**：在工单详情页展示关联的报价单记录列表，点击查看报价单详情

  **报价单制作** (v2.2新增)：
  * **报价单编辑**：支持添加、修改、删除配件明细
  * **配件明细一键导入**：支持Excel模板导入大量配件数据，自动匹配配件编码、名称、规格、单价
  * **报价计算**：自动计算小计、税额、总价，支持折扣设置
  * **报价单模板**：可配置报价单抬头、落款、条款等
  * **成本与利润分析** (v2.2.1新增)：
    - 配件明细显示采购金额（成本）
    - 自动计算每条配件的利润率
    - 报价单底部显示合计：采购金额、销售金额、总利润率

  **报价单审核** (v2.2新增)：
  * **审核流程**：工程师提交 -> 销售主管初审 -> 财务复审 -> 总经理终审（可配置）
  * **审核记录**：记录每次审核意见、审核人、审核时间
  * **审核状态**：待审核/审核中/已通过/已驳回
  * **驳回处理**：被驳回后可修改重新提交

  **报价单PDF生成与预览** (v2.2新增，v2.2.1优化)：
  * **PDF模板**：根据指定Excel模板生成标准报价单格式
  * **电子签章**：审核通过后自动加盖公司电子公章
  * **PDF预览**：审核通过后提供PDF预览功能，可在线查看
  * **下载功能**：支持下载PDF文件
  * **邮件发送**：可直接发送PDF报价单至客户邮箱

  **报价单管理** (v2.2新增)：
  * **报价日期记录**：提交客户时自动记录报价日期
  * **有效期管理**：设置报价有效期，到期自动提醒
  * **版本管理**：支持报价单修改后生成新版本，保留历史版本

  **订单转化** (v2.2新增)：
  * **订单确认**：客户确认订单后，批量维护订单日期
  * **配件型号确认**：逐条确认具体订单的配件型号、数量、价格
  * **库存锁定**：确认订单后锁定相应配件库存
  * **订单追踪**：记录订单状态（待发货/已发货/已完成）
  * **客户回传PDF** (v2.2.1新增)：
    - 客户收到报价单PDF后回传签章版本
    - 职员端可手工上传客户回传的订单PDF
    - 保存客户回传PDF作为订单确认凭证

  **PDF 报告**：内置 HTML 模板引擎，完工后自动填充数据（故障现象、处理过程、更换配件、现场照片等），生成带电子签名的 PDF，支持一键发送给客户。

  **配件销售列表：支持模糊检索，支持多条件筛选，排序，列表显示字段以及显示顺序自定义**

* **FR-4**: 动态二维码安全体系 (Security QR) 

  **核心痛点解决**：防止二维码被提前复制、伪造。

  * **空白码池**：系统预先生成一批加密二维码（内容仅为随机 Token，无业务含义），批量下载打印给工程师。

  * **绑定激活**：

    * 工程师现场安装时，扫码 -> 输入/扫描机器铭牌序列号 -> 系统建立 `Token <-> 机器 ID` 绑定关系。

    * 绑定后，该二维码内容动态跳转至该设备的专属页面。

  * **防伪验证**：客户扫码时，系统校验绑定状态，若发现异常（如多地同时扫码）触发警报。

* **FR-5**: 外勤与日历 (Field Service & Calendar)

  **核心痛点解决**：人员去向不明、工作饱和度无法量化。

  *  **打卡引擎**：

    * **双模式**：

      * **工单打卡**（安装、维修、点检、改造）：必须关联具体工单 ID。

      * **活动打卡**（商务洽谈、客户拜访）：关联客户 ID，用于商谈拜访或无工单的现场勘查等活动。

    * **打卡流程：**选择打卡类型 → 选择客户/工单 → 获取位置 → 拍照/上传 → 提交

    * **防作弊**：调用企业微信/高德定位，限制打卡范围（如客户地址周边 500 米），强制水印相机（时间、地点、人物）。

    * **签离逻辑**：支持"开始 - 结束"闭环，计算工时。

    * **活动内容记录：**对当次客户拜访内容进行记入。

  *  **审批流**：

    配置化工作流引擎。活动内容记录后，工程师自行提交打卡记录，发直属领导审批。

    审批记录留痕，支持驳回重填。

  *  **打卡提醒机制** (v2.3新增)：

    **核心痛点解决**：工程师忘记提交打卡、审批者遗漏审批事项。

    * **未提交提醒**：
      * 工程师完成打卡（签离）后，如未在指定时间内提交审批，系统自动发送提醒
      * 提醒方式：企业微信消息推送、系统内消息通知
      * 提醒频率：可配置（如每天一次、每小时一次）
      * 提醒内容：包含未提交打卡数量、最后打卡时间、快捷提交入口

    * **待审批提醒**：
      * 当有工程师提交打卡记录待审批时，自动通知审批者
      * 提醒方式：企业微信消息推送、系统内消息通知、邮件通知
      * 提醒频率：实时提醒 + 定时汇总提醒（如每天上午9点汇总推送）
      * 提醒内容：包含待审批打卡数量、提交人、提交时间、快捷审批入口

    * **提醒配置**：
      * 管理员可配置提醒规则（开启/关闭、提醒频率、提醒方式）
      * 支持按角色/人员配置不同的提醒策略
      * 支持设置免打扰时段

    * **提醒记录**：
      * 记录所有提醒发送历史
      * 支持查看提醒状态和阅读回执

    **日历视图**：

  - **个人日历**：个人日程计划、查看自己的工单分布、打卡记录。

  - **管理日历**：经理查看团队日程，团队打卡记录，团队分布（地图模式），点击日期下的色块查看工单进度（待派单、进行中、待验收）。

  - 月视图/周视图/日视图切换

  - 颜色区分不同类型活动

  - 点击查看详情

  - 拖拽调整时间

* **FR-8**: 联系人（客户员工）登录与注册体系 (Contact Login & Registration)

  **核心痛点解决**：客户员工需要便捷的入口访问系统，无需记忆复杂密码。

  * **访问入口**：

    * **扫码访问（主要入口）**：
      * 客户扫描设备上的动态二维码
      * 系统识别设备Token，解析设备信息
      * 检查客户登录状态：
        * 已登录 → 直接显示设备详情页
        * 未登录 → 跳转注册/登录页面（保存目标页面，登录后自动跳转）
      * 实现：每个设备有唯一的动态二维码，扫码后访问 `/asset-detail?token=xxx`
    
    * **微信服务号菜单**：
      * 关注企业微信服务号
      * 点击底部菜单"客户服务"：
        * 子菜单：注册/登录 → 跳转 `/contact-register`
        * 子菜单：我的设备 → 未登录则跳转 `/contact-register`
        * 子菜单：发起工单 → 未登录则跳转 `/contact-register`
        * 子菜单：服务进度 → 未登录则跳转 `/contact-register`
      * 实现：服务号菜单配置URL `https://系统域名/contact-register`，所有功能页面都需检查登录状态
    
    * **短信/邮件邀请链接**：
      * 业务员在系统中点击"邀请客户"
      * 输入客户手机号/邮箱
      * 系统生成带Token的邀请链接
      * 发送短信/邮件给客户
      * 客户点击链接 → 跳转 `/contact-register?invite=xxx&phone=138xxxx`
      * 自动填充手机号，简化注册流程
      * 邀请链接格式：Token有效期7天，一个Token只能使用一次
    
    * **直接访问**：
      * PC端/移动端直接访问 `https://系统域名/contact-register`
      * 适用场景：浏览器直接访问、公司官网链接、其他渠道获取的注册链接

  * **登录方式（注册成功后）**：

    * **手机号+短信验证码（推荐）**：
      * 访问系统任意页面 → 未登录则显示登录页面
      * 输入手机号 → 点击"获取验证码" → 输入6位短信验证码 → 登录成功
      * 特点：无需记忆密码、每次登录需验证码、安全性高、适合不经常访问的客户
      * 实现页面：`/customer-login`
    
    * **微信免密登录（最便捷）**：
      * 在微信/企业微信内打开系统
      * 自动获取微信OpenID
      * 系统匹配已注册账号：已注册→自动登录，未注册→跳转注册页面
      * 特点：无需输入任何信息、一键登录、体验最佳
      * 实现：调用微信OAuth授权，获取用户OpenID和基本信息，自动匹配或创建账号
    
    * **Token自动登录（临时访问）**：
      * 使用场景：查看工单进度、确认服务报告、查看设备状态
      * 系统发送带Token的链接给客户（短信/邮件/微信）
      * 客户点击链接 → 验证Token有效性 → 有效则自动登录并跳转目标页面
      * 特点：无需注册即可查看特定信息、Token有效期短（默认7天）、权限受限

  * **访问权限控制**：

    * **注册前（未登录）**：
      * ✅ 访问注册页面 `/contact-register`
      * ✅ 访问登录页面 `/customer-login`
      * ❌ 查看设备信息、发起工单、查看服务进度
    
    * **注册后待审核**：
      * ✅ 访问注册页面（显示"审核中"状态）
      * ✅ 查询审核进度
      * ❌ 发起工单、查看设备信息
    
    * **审核通过后**：
      * ✅ 查看所属企业设备、发起服务工单、查看工单进度
      * ✅ 查看服务历史、下载服务报告、评价服务

  * **注册流程（天眼查API搜索选择 + 员工审核）**：

    **核心设计原则**：
    * **完全防重复**：客户无法手填企业名称，必须从天眼查标准数据库选择
    * **数据标准化**：所有企业信息来自天眼查，确保数据质量和一致性
    * **简化注册流程**：搜索选择企业 → 自动填充信息 → 填写个人信息 → 提交审核

    * **步骤1 - 从天眼查搜索企业**：
      * **输入关键词**：客户输入企业名称关键词（如"腾讯"、"阿里巴巴"）
      * **调用API搜索**：点击搜索按钮，系统调用天眼查API进行企业搜索
      * **显示搜索结果**：展示天眼查返回的企业列表，每个企业显示：
        * 企业全称
        * 统一社会信用代码
        * 注册地址
        * 法人代表
        * 企业状态（存续/在业/注销等）
      * **选择企业**：客户从列表中选择正确的企业
      * **备用方案**：如未找到企业，提供"手动输入"选项（需更严格审核）
      * **API集成细节**：
        * 接口：天眼查企业搜索接口（/services/v3/search/sNorV2）
        * 搜索方式：支持关键词模糊搜索
        * 返回字段：企业名称、统一社会信用代码、注册地址、法人代表、企业状态、注册资本、成立日期等
        * 数据缓存：本地缓存搜索结果，避免重复查询消耗API额度
        * 错误处理：API调用失败时提供友好提示，引导手动输入
    
    * **步骤2 - 确认企业信息**：
      * **自动填充**：系统根据所选企业自动填充全部信息
      * **信息展示**：以卡片形式展示已选企业信息：
        * 企业名称
        * 统一社会信用代码
        * 注册地址
        * 法人代表
        * 企业状态
      * **重新选择**：提供"重新选择"功能，允许客户返回重新搜索
    
    * **步骤3 - 填写个人信息**：
      * 姓名（必填）：申请人真实姓名
      * 手机号（必填）：用于接收验证码和审核通知，作为登录账号
      * 职位（必填）：申请人在企业中的职位
      * 邮箱（选填）：用于接收系统通知
    
    * **步骤4 - 身份验证**：
      * **手机号短信验证码**：
        * 输入手机号 → 点击"获取验证码"
        * 接收6位数字验证码（5分钟有效）
        * 输入验证码完成验证
      * **防刷机制**：
        * 同一手机号1分钟内只能发送1次验证码
        * 每小时最多发送10次验证码
        * 验证码错误5次后需等待30分钟
      * **微信授权（可选）**：微信/企业微信内打开 → 授权获取OpenID → 绑定手机号
    
    * **步骤5 - 提交注册**：
      * 阅读并同意《用户注册协议》
      * 提交注册申请
      * 创建"待审核"客户账号（状态：pending）
      * 如企业未在系统中存在，同时创建"待审核"企业档案（状态：pending）
      * 生成注册申请单，进入审核队列
      * 发送审核通知给业务员/管理员（企业微信/短信/系统消息）
      * 客户收到"审核中"状态提示，可查看审核进度
    
    * **防重复机制（终极方案）**：
      * **天眼查数据源**：所有企业来自天眼查标准数据库，数据权威准确
      * **统一信用代码唯一性**：每个企业有唯一的18位统一社会信用代码
      * **无手填入口**：客户无法随意输入企业名称，必须从列表选择
      * **手动输入备用**：仅当天眼查未收录时才允许手动输入，且需要更严格的审核流程
      * **系统内重复检测**：即使从天眼查选择，仍需检查该企业是否已在系统内注册

  * **员工审核流程**：
    * **审核任务分配**：
      * 系统自动分配给对应区域的业务员（根据企业地址区域）
      * 或统一进入"客户注册审核"待办池，由客服专员分配
    
    * **审核内容（简化版）**：
      * **企业信息审核**：
        * 核对企业是否已在系统内存在（通过统一社会信用代码）
        * 如已存在，直接关联到已有企业，无需重复创建
        * 如为新企业，自动创建企业档案（数据来自天眼查，无需人工核对）
      * **联系人身份审核**：
        * 确认申请人为企业员工（通过电话核实或邮箱域名验证）
        * 核实职位真实性
      * **手动录入企业特殊审核**：
        * 如客户使用手动输入（非天眼查数据），需额外审核：
          * 要求上传营业执照
          * 核对企业名称和信用代码准确性
          * 审核时间延长至5个工作日
    
    * **审核操作**：
      * **通过**：
        * 企业和客户账号同时激活（状态：active）
        * 如为新企业，自动创建企业档案
        * 发送通知给客户（短信+微信）
        * 自动分配专属业务员跟进
      * **拒绝**：
        * 填写拒绝原因（如"信息不实"、"非目标客户"等）
        * 客户可修改后重新提交
      * **要求补充信息**：
        * 退回给客户补充材料（主要针对手动录入的企业）
        * 如"请上传营业执照"、"请补充详细地址"等
    
    * **审核时效**：
      * 天眼查数据源：建议1-2个工作日内完成审核（数据已标准化，审核简单）
      * 手动录入企业：建议3-5个工作日内完成审核（需要额外核实）
      * 超期自动提醒审核人

  * **审核通过后**：
    * 客户收到短信+微信通知："您的注册申请已通过审核，现在可以登录系统"
    * 客户可登录系统查看所属企业的设备、发起工单、查看历史记录
    * 系统自动分配专属业务员跟进，业务员收到新客户分配通知
    * 客户状态变为"正常"，企业状态变为"合作中"

  * **登录方式**：

    * **手机号免密登录**：输入手机号 -> 接收短信验证码 -> 验证通过即登录
    
    * **微信免密登录**：微信/企业微信内打开 -> 自动获取OpenID -> 匹配已绑定账号 -> 直接登录
    
    * **Token自动登录**：通过带Token的URL访问 -> 验证Token有效性 -> 自动登录并跳转目标页面

  * **权限控制**：

    * **注册前**：只能访问注册页面，无法查看任何系统数据
    
    * **待审核**：注册提交后，提示"审核中"，无法发起工单，可查看审核进度
    
    * **审核通过后**：可查看所属企业的设备、发起工单、查看历史记录
    
    * **账号停用**：如企业合作终止，账号变为"停用"状态，仅可查看历史数据，不可发起新工单

  * **安全机制**：

    * Token有效期控制（默认7天）
    
    * 短信验证码5分钟有效期
    
    * 同一手机号短信发送频率限制（1分钟/次，10次/小时）
    
    * 设备更换或异常登录提醒
    
    * 密码策略（如设置密码）：8位以上，包含大小写字母和数字

    | 工程师 | ✅ | ✅ | 可自由切换 |
    | 技术部长 | ✅ | ✅ | 可自由切换 |
    | 业务员 | ✅ | ❌ | 无切换按钮 |

    * **删除限制**：有下级部门或人员的部门不可删除
    * **历史保留**：已停用部门保留历史数据，可查看历史人员

  * **部门人员管理**：

* **FR-11**: 员工账号管理 (Employee Account Management) (v2.5新增)

  **核心痛点解决**：员工账号管理分散，入职离职流程不规范，权限分配混乱。

  * **员工账号生命周期**：
    * **创建账号**：为新员工创建系统账号，自动生成初始密码
    * **编辑信息**：修改员工基本信息、联系方式、职位等
    * **停用账号**：员工离职时停用账号，保留历史数据
    * **删除账号**：彻底删除账号（仅超级管理员可操作）
    * **批量导入**：支持Excel批量导入员工信息

  * **员工信息管理**：
    * **基本信息**：姓名、工号、性别、手机号、邮箱、入职日期
    * **职位信息**：职位、职级、部门、直属上级
    * **账号信息**：登录账号、账号状态、最后登录时间
    * **扩展信息**：身份证号、紧急联系人、地址等（可选）
    * **头像管理**：上传员工头像，支持默认头像

  * **角色与权限分配**：
    * **角色绑定**：为员工分配系统角色（工程师/技术部长/业务员/系统管理员）
    * **数据权限**：设置员工的数据访问范围（全部/本部门/仅自己）
    * **功能权限**：基于角色的功能权限自动分配
    * **权限继承**：部门负责人自动继承部门管理权限

  * **账号安全管理**：
    * **密码策略**：强制密码复杂度（8位以上，包含大小写字母和数字）
    * **密码重置**：管理员可重置员工密码，员工首次登录需修改
    * **登录锁定**：连续5次登录失败锁定账号30分钟
    * **密码过期**：密码90天强制更换（可配置）

  * **员工状态管理**：
    * **在职**：正常在职员工，可正常使用系统
    * **试用**：试用期员工，权限可单独配置
    * **离职**：已离职员工，账号停用但保留数据
    * **锁定**：因安全原因临时锁定

  * **入职/离职流程**：
    * **入职登记**：记录入职日期、试用期、合同信息等
    * **离职办理**：记录离职日期、离职原因、工作交接情况
    * **账号交接**：离职员工工单、客户自动交接给指定人员
    * **数据归档**：离职员工数据归档，支持历史查询

  * **登录日志**：
    * **登录记录**：记录每次登录的时间、IP地址、设备信息
    * **操作日志**：记录关键操作（修改密码、权限变更等）
    * **异常登录**：检测异常登录行为并提醒
    * **日志查询**：支持按时间、员工、操作类型查询日志

* **FR-12**: 客户注册审核管理 (Customer Registration Review) (v2.6新增)

  **核心痛点解决**：客户自主注册后，需要员工审核通过才能使用系统，避免垃圾注册和重复企业数据。

  * **审核工作台**：
    * **待审核列表**：展示所有待审核的客户注册申请
      * 显示字段：申请时间、企业名称、联系人姓名、手机号、申请来源（扫码/微信菜单/URL）
      * 排序：默认按申请时间倒序，支持按时间、企业名称排序
      * 筛选：按申请时间范围、企业名称关键词、审核状态筛选
    * **审核统计看板**：
      * 今日待审核数量
      * 本周待审核数量
      * 超期未审核数量（超过3个工作日）
      * 审核通过率统计
    * **快速操作**：支持批量通过、批量拒绝、批量分配审核人

  * **审核详情页**：
    * **企业信息展示**：
      * 企业名称、统一社会信用代码、企业地址
      * **智能提示**：显示相似企业警告（相似度>70%时高亮提示）
      * **天眼查信息**：如调用了天眼查API，显示标准企业信息供对比
      * **信用代码校验**：显示格式校验结果和是否已存在
    * **联系人信息展示**：姓名、手机号、邮箱、职位、部门
    * **注册信息展示**：申请时间、申请来源、IP地址、设备信息
    * **历史记录**：该手机号/邮箱是否有过注册记录

  * **审核操作**：
    * **通过**：
      * 企业和客户账号同时激活（状态变为active）
      * 自动分配专属业务员（根据企业地址区域或轮询分配）
      * 发送通知给客户（短信+微信）："您的注册申请已通过审核"
      * 发送通知给业务员（企业微信）："您有新分配的客户XXX"
      * 记录审核日志：审核人、审核时间、审核结果
    * **拒绝**：
      * 填写拒绝原因（预设选项：信息不实、非目标客户、重复申请、其他）
      * 填写拒绝说明（详细原因）
      * 发送通知给客户："您的注册申请未通过审核，原因：XXX"
      * 客户可修改信息后重新提交
    * **要求补充信息**：
      * 选择需要补充的信息项（营业执照、详细地址、企业官网、其他）
      * 填写补充说明
      * 退回给客户，状态变为"待补充"
      * 客户补充后重新进入审核队列
    * **合并到已有企业**：
      * 发现重复企业时，选择已有企业进行合并
      * 将新客户关联到已有企业
      * 废弃新建企业申请
      * 发送通知给客户："您已加入已有企业XXX"

  * **审核分配机制**：
    * **自动分配**：
      * 根据企业地址区域自动分配给对应区域业务员
      * 或按轮询方式平均分配给客服团队
    * **手动分配**：
      * 审核专员可将申请手动分配给指定业务员
      * 支持批量分配
    * **审核人变更**：
      * 支持审核人转交申请给其他同事
      * 记录转交历史

  * **审核提醒机制**：
    * **超期提醒**：申请超过3个工作日未审核，自动提醒审核人和上级
    * **新申请提醒**：有新申请提交时，实时通知审核人（企业微信/短信）
    * **每日汇总**：每天下班前发送待审核汇总给审核团队

  * **企业信息管理（审核后）**：
    * **企业档案**：审核通过后自动生成企业档案
    * **企业分级**：支持设置企业等级（VIP/普通/潜在）
    * **企业状态**：合作中/暂停合作/终止合作
    * **企业编辑**：支持修改企业信息（需权限控制）
    * **企业合并**：发现重复企业时，支持合并企业数据和客户关联

  * **审核日志与查询**：
    * **审核日志**：记录所有审核操作（通过/拒绝/补充/合并）
    * **操作人追踪**：记录审核人、审核时间、审核意见
    * **历史查询**：支持按企业、联系人、审核人、时间范围查询审核历史
    * **数据统计**：审核通过率、平均审核时长、审核量排名等

## Non-Functional Requirements

* **NFR-1**: 性能要求

  * 系统响应时间不超过2秒

  * 支持至少100人同时在线使用

  * 移动端离线操作后数据同步时间不超过5分钟

* **NFR-2**: 安全要求

  * 数据加密存储

  * 基于角色的访问控制

  * 操作日志记录

  * 符合企业微信安全规范

* **NFR-3**: 可用性要求

  * 系统可用性达到99.9%

  * 定期数据备份

  * 灾难恢复机制

* **NFR-4**: 可扩展性要求

  * 模块化设计

  * 支持未来功能扩展

  * 支持数据量增长

* **NFR-5**: 用户体验要求

  * 界面美观易用

  * 操作流程简洁明了

  * 移动端适配不同设备

### AC-8: 联系人登录与注册功能 (v2.6最终版)

* **Given**: 联系人通过扫码/URL/微信菜单访问系统

* **When**: 联系人完成自主注册并通过员工审核

* **Then**: 能够通过手机号短信或微信免密登录，查看所属企业设备、发起工单

* **Verification**: `programmatic`

* **Notes**: 
  * **访问入口**：
    * **扫码访问**：扫描设备二维码 → 未登录则跳转注册页面 → 登录后自动返回设备详情页
    * **微信服务号菜单**：点击菜单项 → 未登录则跳转注册页面 → 登录后进入功能页面
    * **短信/邮件邀请链接**：点击邀请链接 → 自动填充手机号 → 简化注册流程
    * **直接访问**：通过浏览器直接访问注册页面
  * **客户自主注册流程（天眼查API搜索选择方案）**：
    * **步骤1 - 从天眼查搜索企业**：输入关键词 → 调用天眼查API → 显示企业列表 → 选择企业
    * **步骤2 - 确认企业信息**：系统自动填充企业信息（名称、信用代码、地址、法人、状态）
    * **步骤3 - 填写个人信息**：姓名、手机号、职位、邮箱（选填）
    * **步骤4 - 身份验证**：手机号短信验证码（6位，5分钟有效，防刷机制）
    * **步骤5 - 提交注册**：同意协议 → 提交申请 → 创建待审核账号
  * **天眼查API集成**：
    * 接口：天眼查企业搜索接口（/services/v3/search/sNorV2）
    * 返回字段：企业名称、统一社会信用代码、注册地址、法人代表、企业状态等
    * 数据缓存：本地缓存避免重复查询
    * 备用方案：天眼查未收录时允许手动输入（需更严格审核）
  * **防重复机制（终极方案）**：
    * 完全防重复：客户无法手填企业名称，必须从列表选择
    * 天眼查数据源：所有企业来自天眼查标准数据库
    * 统一信用代码唯一性：每个企业有唯一的18位信用代码
    * 手动输入备用：仅当天眼查未收录时允许，需更严格审核
  * **登录方式（注册成功后）**：
    * **手机号+短信验证码**：输入手机号 → 获取验证码 → 登录成功 → 跳转到原目标页面
    * **微信免密登录**：微信内打开 → 自动获取OpenID → 匹配账号 → 自动登录
    * **Token自动登录**：点击带Token链接 → 验证有效性 → 自动登录 → 跳转目标页面
  * **访问权限控制**：
    * 注册前：只能访问注册/登录页面
    * 待审核：显示"审核中"状态，可查询进度
    * 审核通过：可查看设备、发起工单、查看进度等全部功能
  * **审核前限制**：注册提交后状态为"待审核"，无法发起工单，可查看审核进度
  * **审核通过后**：收到短信+微信通知，账号激活，自动分配专属业务员
  * **安全机制**：短信验证码5分钟有效，防刷机制（1分钟/次，10次/小时）

### AC-11: 客户注册审核管理功能 (v2.6最终版)

* **Given**: 业务员/客服专员已登录

* **When**: 进入客户注册审核模块

* **Then**: 能够查看待审核列表、审核客户注册申请、管理企业信息

* **Verification**: `programmatic`

* **Notes**: 
  * **审核工作台**：
    * 待审核列表展示：申请时间、企业名称（标记天眼查/手动）、联系人、手机号、数据来源
    * 审核统计看板：今日/本周待审核、超期数量、通过率、天眼查/手动录入占比
    * 支持批量操作：批量分配、批量通过（仅限天眼查数据）
  * **审核详情页**：
    * 企业信息展示：名称、信用代码、地址、法人、状态（天眼查数据直接展示，无需核对）
    * 数据来源标记：明确标记"天眼查数据"或"手动录入"
    * 系统内重复检测：检查该企业是否已在系统内存在（通过信用代码）
    * 联系人信息：姓名、手机号、职位、邮箱
    * 注册来源：扫码/微信菜单/URL
  * **审核内容（简化版）**：
    * 天眼查数据源：主要审核联系人身份，企业信息已标准化
    * 手动录入企业：需额外审核营业执照、核对企业信息准确性
  * **审核操作**：
    * **通过**：激活账号、创建企业档案（如为新企业）、分配业务员、发送通知
    * **拒绝**：填写原因、客户可重新提交
    * **要求补充信息**：主要针对手动录入企业，要求补充材料
  * **审核时效**：
    * 天眼查数据源：1-2个工作日（审核简单）
    * 手动录入企业：3-5个工作日（需额外核实）
    * 超期提醒：自动提醒审核人
  * **企业信息管理**：
    * 审核通过后自动生成企业档案
    * 支持企业分级（VIP/普通/潜在）
    * 手动录入企业支持后续补录天眼查信息
  * **审核日志**：记录所有审核操作、支持历史查询、审核数据统计
