# 客户访问注册页面和系统流程说明

## 一、客户如何进入注册页面

### 入口1：设备二维码扫描（主要入口）

**流程：**
```
客户扫描设备上的动态二维码
    ↓
系统识别设备Token → 解析设备信息
    ↓
检查客户登录状态
    ├─ 已登录 → 直接显示设备详情页
    └─ 未登录 → 跳转登录/注册页面
```

**实现：**
- 每个设备有唯一的动态二维码
- 扫码后访问 `/asset-detail?token=xxx`
- 系统判断登录状态，未登录则重定向到 `/contact-register`

---

### 入口2：微信服务号菜单

**流程：**
```
关注企业微信服务号
    ↓
点击底部菜单"客户服务"
    ├─ 子菜单：注册/登录 → 跳转 `/contact-register`
    ├─ 子菜单：我的设备 → 未登录则跳转 `/contact-register`
    ├─ 子菜单：发起工单 → 未登录则跳转 `/contact-register`
    └─ 子菜单：服务进度 → 未登录则跳转 `/contact-register`
```

**实现：**
- 服务号菜单配置URL：`https://系统域名/contact-register`
- 所有功能页面都需检查登录状态

---

### 入口3：短信/邮件邀请链接

**流程：**
```
业务员在系统中点击"邀请客户"
    ↓
输入客户手机号/邮箱
    ↓
系统生成带Token的邀请链接
    ↓
发送短信/邮件给客户
    ↓
客户点击链接 → 跳转 `/contact-register?invite=xxx`
    ↓
自动填充手机号 → 简化注册流程
```

**实现：**
- 邀请链接格式：`/contact-register?invite=TOKEN&phone=138xxxx`
- Token有效期：7天
- 一个Token只能使用一次

---

### 入口4：直接访问注册页面

**访问地址：**
- PC端：`https://系统域名/contact-register`
- 移动端：`https://系统域名/contact-register`

**适用场景：**
- 客户通过浏览器直接访问
- 从公司官网点击"客户登录"链接
- 从其他渠道获取的注册链接

---

## 二、注册成功后的访问方式

### 方案一：手机号+短信验证码（推荐）

**登录流程：**
```
访问系统任意页面
    ↓
未登录 → 显示登录页面
    ↓
输入手机号
    ↓
点击"获取验证码"
    ↓
输入6位短信验证码
    ↓
登录成功 → 跳转到原目标页面
```

**特点：**
- 无需记忆密码
- 每次登录都需要验证码
- 安全性高
- 适合不经常访问的客户

**实现页面：** `/customer-login`

---

### 方案二：微信免密登录（最便捷）

**登录流程：**
```
在微信/企业微信内打开系统
    ↓
自动获取微信OpenID
    ↓
系统匹配已注册账号
    ├─ 已注册 → 自动登录
    └─ 未注册 → 跳转注册页面
```

**特点：**
- 无需输入任何信息
- 一键登录
- 体验最佳
- 适合微信生态内的客户

**实现：**
- 调用微信OAuth授权
- 获取用户OpenID和基本信息
- 自动匹配或创建账号

---

### 方案三：Token自动登录（临时访问）

**使用场景：**
- 查看工单进度
- 确认服务报告
- 查看设备状态

**流程：**
```
系统发送带Token的链接给客户（短信/邮件/微信）
    ↓
客户点击链接
    ↓
验证Token有效性
    ├─ 有效 → 自动登录并跳转目标页面
    └─ 过期 → 提示重新获取链接
```

**特点：**
- 无需注册即可查看特定信息
- Token有效期短（默认7天）
- 权限受限（只能查看指定内容）
- 适合一次性访问场景

## 三、客户访问权限控制

### 注册前（未登录）
- ✅ 访问注册页面 `/contact-register`
- ✅ 访问登录页面 `/customer-login`
- ❌ 查看设备信息
- ❌ 发起工单
- ❌ 查看服务进度

### 注册后待审核
- ✅ 访问注册页面（显示"审核中"状态）
- ✅ 查询审核进度
- ❌ 发起工单
- ❌ 查看设备信息

### 审核通过后
- ✅ 查看所属企业设备
- ✅ 发起服务工单
- ✅ 查看工单进度
- ✅ 查看服务历史
- ✅ 下载服务报告
- ✅ 评价服务

## 四、技术实现要点

### 1. 路由守卫（Router Guard）

```javascript
// router/index.js
router.beforeEach((to, from, next) => {
  // 公开页面，无需登录
  const publicPages = ['/contact-register', '/customer-login', '/scan']
  if (publicPages.includes(to.path)) {
    return next()
  }
  
  // 检查登录状态
  const isLoggedIn = checkCustomerLogin()
  if (!isLoggedIn) {
    // 保存目标页面，登录后跳转
    sessionStorage.setItem('redirectUrl', to.fullPath)
    return next('/contact-register')
  }
  
  // 检查审核状态
  const customerStatus = getCustomerStatus()
  if (customerStatus === 'pending') {
    return next('/contact-register?status=pending')
  }
  
  next()
})
```

### 2. 登录状态存储

```javascript
// 使用localStorage存储登录信息
```

```javascript
// 微信登录流程
const wechatLogin = () => {
  // 1. 获取微信授权码

  const authUrl = `https://open.weixin.qq.com/connect/oauth2/authorize?appid=${APPID}&redirect_uri=${redirectUri}&response_type=code&scope=snsapi_base&state=STATE#wechat_redirect`
  
  // 2. 跳转微信授权页
  window.location.href = authUrl
  
  // 3. 回调处理
  // 获取code → 换取access_token → 获取openid → 匹配账号
}
```

### 4. Token自动登录

```javascript
// URL带Token的自动登录
const tokenLogin = (token) => {
  // 验证Token
  const valid = verifyToken(token)
  if (valid) {
    // 设置临时登录状态
    sessionStorage.setItem('tempToken', token)
    // 跳转到目标页面
    router.push(targetUrl)
  } else {
    // Token过期，提示重新获取
    ElMessage.error('链接已过期，请联系业务员重新发送')
  }
}
```
